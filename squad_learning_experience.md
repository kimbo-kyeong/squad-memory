## 2026-06-26 업데이트: 학습 유저 비율 정의 변경 (코호트 aging → 현재 주 단면)

**기존**: 코호트(수강시작일) 기준 "N주차에 레슨 1개+ 완료 수강생 / 전체 수강생".
**변경(확정)**: **현재 주 단면(cross-sectional)** 으로 재정의.
- **모수** = 이번 주에 코스가 진행 중인 수강생 = `course_week_now ∈ [1, total_weeks]`. **수료자·취소자 제외**.
  - `course_week_now = FLOOR(DATEDIFF(day, 개강일, 오늘)/7)+1` (KST, 개강+9h). 26주차 개강=코스1주차, 25주차 개강=코스2주차 …
  - `total_weeks = CEIL(product.duration_days/7.0)`
- **분자(학습유저)** = 자기 **현재 코스주차 윈도우**에 ≥1강 통과 = `FLOOR(DATEDIFF(day,개강일,passed_date)/7)+1 == course_week_now` (lesson_record.is_passed='true'). "최근 7일"(느슨) 아님 — 코스주차 기준(엄격).
- **검증(2026-06-26, 현재 26주차)**: 전체 모수 922 / 학습유저 338 = **36.7%**. 코스주차별 39.6%→37.3%→30.9%→28.8%→27.3% 단조 감소(후반 이탈).
- **SQL 함정**: `date + FLOOR()/CEIL() 결과(소수형)` 산술은 매칭 0건 버그 → 윈도우를 날짜덧셈 말고 **코스주차 인덱스 동치 비교**로 구현. 모든 주차 정수 캐스팅(`::int`).
- 이 모수 정의(코스 진행중·수료자제외·코스주차별) = **고객 세분화 코호트 작업의 scope·시간동학 'active' 정의와 동일** (같은 모집단 위 정렬). 코호트 진도축은 `enrollment.progress_rate`(영상+실습 통합, 수료판정 일치) 사용, 수경스 영상-진도(준수율, 실습제외)와 다른 지표임에 유의. [[kdc_completion_vs_practice]]
