### 주의할 패턴·해석 가이드
- 진행 중 주차는 부분 데이터. 단순 비교 시 -50% 이상 떨어진 것처럼 보이지만 노이즈
- 리드와 결제시작은 다른 액션. 페이지 방문(리드) vs 결제하기 버튼 클릭(결제시작) → 리드 > 결제시작이 정상
- 리드는 기간 기준, 나머지는 코호트 기준
- KDC 다구매율 약 1.16 — (user, product) 코호트 사용 시 user 단위보다 16% 매출 정합 영향
- TOP5 과목 신규 회차 오픈 직후 첫 주는 노이즈 큼
- KDT 세일즈 분리는 is_sales_target = true 기준 (is_sales_applied 아님)

### KDC grain 정책 (2026-05-07 최신)
- KPI 시트 = (user, product) 코호트 12컬럼 + 누적 7컬럼
- 누적 컬럼 (모든 행 동일 값): 2026_누적_결제시작/완료/등록/매출 + 2026_누적_유저_결제시작/완료/등록
- 상세 시트 = user 단위 발생 기준 — 주차별 트렌드 비교, segment 분리 (40대/TOP5)
- 결제완료 = amplitude `sc_payment_purchase_completed`, DB `order_fin_at`은 검증 컬럼
- 카드조회 = amplitude 이벤트엔 product_id 없어 user-level 귀속 → 다구매자는 product 수만큼 + (참고용)

## 구현 메타데이터 (참고용)

### 쿼리 파일
| 파일 | 용도 | 코호트/발생 |
|------|------|------------|
| `그로스_KDT_KPI시트.sql` | 제품실 KPI 시트 KDT Raw | 코호트 (서류합격 주차), 세일즈/비세일즈 행 분리 |
| `그로스_KDC_KPI시트.sql` | 제품실 KPI 시트 KDC + 그로스KDC_raw | (user, product) 코호트 (첫 클릭 주차), 리드는 기간 기준 |
| `그로스_KDT_트랙별_퍼널.sql` | 그로스 상세 시트 KDT | 발생 기준 |
| `그로스_KDC_과목별_퍼널.sql` | 그로스 상세 시트 KDC | 발생 기준 weekly |

### KDT 상세 시트 설계 (2026-04-23 신규)
- 그래뉼래리티: 트랙 × 회차 × 모집 상대주차 (코호트 기준, 누적)
- 모집 시작~종료일자를 week로 나눠서 "모집1주차, 2주차, ..." 표현
- 퍼널: 서류합격 → 결제시작 → 결제완료 → (수강신청완료/확인중) → 세일즈타겟수(성공/진행중/실패, 세일즈성공률) → 선발완료 → 등록
- 추가 지표: 리드→등록 소요시간 분포 Q25/Q50/Q75 (주차별)

### KDC 상세 시트 설계 (2026-04-23 신규)
- 주차: 화~월 (수강경험 스쿼드와 동일)
- 퍼널: 결제페이지 PV → 카드사 선택 → 결제하기 클릭(시작) → (PG창발생/확인중) → 결제완료 → (수강신청/확인중) → 선발 → 등록
- 리드타임: PG창발생→결제완료, 결제완료→수강신청
- 세그먼트별 분리: 전체 / 40대이상·40대미만 / TOP5·비TOP5

### 라벨 규칙
- 주차 라벨: `2026-01W` (리딩제로 포함)
- KDC business 필터: event_properties.business = 'kdcScc'
- KPI 시트는 볼륨만 출력, 전환율은 Excel 누적 합산으로 계산
