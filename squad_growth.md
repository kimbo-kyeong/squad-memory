## KDC grain 정책 (2026-05-07 최신)

- **KPI 시트** = (user, product) 코호트 12컬럼 + 누적 7컬럼 (행 아닌 컬럼으로)
  - 주간 행: 깔끔하게 (user, product) 코호트만
  - 누적 컬럼 (모든 행 동일 값): 2026_누적_결제시작/완료/등록/매출 (4) + 2026_누적_유저_결제시작/완료/등록 (3)
  - UNION ALL 누적 행 → 컬럼 변경 사유: SQL 클라이언트가 NULL/빈값 섞인 varchar 컬럼을 epoch ms로 변환하는 이슈
  - 다구매율 1.16 (4,645 유저 → 5,369 (user, product), 16% 매출 정합 영향)
- **상세 시트** = user 단위 발생 기준 — 주차별 트렌드 비교, segment 분리 (40대/TOP5)
- **결제완료** = amplitude `sc_payment_purchase_completed` (이벤트명 검증 필요), DB `order_fin_at`은 검증 컬럼으로 끝에 노출
- **카드조회** = amplitude 이벤트엔 product_id 없어 user-level 귀속 → 다구매자는 product 수만큼 + (참고용)

## 주차 기준

| 시트 | 사업 | 주차 |
|------|------|------|
| KPI 시트 | KDT | 월~일 (ISO week) |
| KPI 시트 | KDC | 화~월 (개강일 기준) |
| 상세 시트 | KDT | **모집시작 기준 상대주차** (모집1주차, 2주차, ...) |
| 상세 시트 | KDC | 화~월 |

## KDT 상세 시트 설계 (2026-04-23 신규)

**그래뉼래리티**: 트랙 × 회차 × 모집 상대주차 (코호트 기준, 누적)
- 모집시작~종료일자를 week로 나눠서 "모집1주차, 2주차, ..." 로 표현
- 어떤 요일에 모집 시작해도 같은 흐름으로 볼 수 있는 구조

**퍼널**:
서류합격 → 결제시작 → 결제완료 → (*수강신청완료, 확인중) → 세일즈타겟수(성공COMPLETED/진행중/실패LOST, 세일즈성공률) → 선발완료 → 등록

**추가 지표**:
- 리드→등록 소요시간 분포: Q25, Q50, Q75 (주차별)
- 볼륨 + 전환율 모두 구현

## KDC 상세 시트 설계 (2026-04-23 신규)

**주차**: 화~월 (수강경험 스쿼드와 동일)

**퍼널**:
- 모니터링용: 결제페이지 PV, 카드사 선택 버튼
- 결제하기 버튼 클릭(시작) → (*PG창발생, 확인중) → 결제완료 → (*수강신청, 확인중) → 선발 → 등록

**리드타임**:
- PG창발생→결제완료 리드타임
- 결제완료→수강신청 리드타임

**세그먼트별 분리**:
- 전체
- 연령별: 40대이상 / 40대미만 (2026-04-24 변경, 이전 기준: 50/비50)
- 과목별: TOP5 / 비TOP5

## 핵심 주의사항

- 리드와 결제시작은 **다른 액션** (페이지 방문 vs 버튼 클릭)
- 리드는 기간 기준, 나머지는 코호트 기준 → 리드 > 결제시작이어야 정상
- KPI 시트는 볼륨만 출력, 전환율은 Excel 누적 합산으로 계산
- 라벨: `2026-01W` (리딩제로 포함)
- KDT 세일즈 분리: is_sales_target = true 기준 (is_sales_applied 아님)
- KDC business 필터: event_properties.business = 'kdcScc'
